Now -YOU- can control your ASP.NET MVC Errors (again)!

There's a lot going wrong with ASP.NET MVC and the stock standard error handling.

It's broken.

Out of the box:

  • It does a 302 redirect to an error page :sad:
  • It doesn't work with cshtml (Razor) views. Only Html (er, hello 1990?)
  • It's a mess to try and manually hook up and wire up all the extensibility points to handle most scenarios

As I said : PITA, broken, sad panda's.

But NuGet this package and you'll be happy hipsters once more!

Or .. clone/fork this bad boy and help make something that is awesome become fully sick awesome!

Why would I want to do that? What does this do?

You can now create your own .cshtml files (with yummy yummy razor goodness) to handle any Http Status error code!

  • 500 Internal Errors
  • 404 Resource Not Found Errors
  • 401 Not Authorized Errors
  • 418 I'm a TeaPot (if you really care)

Because the error pages you make are just razor pages, you can have whatever magic fairy dust you want in there. switch statements? Sure! if checks? If that floats your boat! (<-- see what I did there?)
